---
title: "Check API Key"
api: "GET http://api.rectify.international/v1/authentication/checkapikey"
description: "This endpoint checks if the apiKey provided is valid."
authMethod: "none"
---

### Body

There is infact a body of this request.

<ParamField body="apiKey" type="string" default="123abc" required>
  The apiKey of the desired request, you must fetch an apiKey in the discord before you can do any requests in this endpoint.
</ParamField>

### Response

<ResponseField name="status" type="number">
  Indicates whether the call was successful. 200 if successful, no response if not.
</ResponseField>

<ResponseField name="data" type="object">

Data object containing the user's information.
</ResponseField>

<Expandable title="Response Info">

<ResponseField name="version" type="string">
  This is the version the API is running on, usually would be PROD(version as an integer).
</ResponseField>

<ResponseField name="success" type="boolean">
  This responds whether the request was successful or not. If the user has been verified before and is able to have their info displayed, it is a boolean value.
</ResponseField>

<ResponseField name="message" type="string">
  This is the message that tells if the apiKey is valid or not. Example: 'Valid API key.'
</ResponseField>

</Expandable>
<RequestExample>

```bash Example Request
curl --location --request GET 'https://api.rectify.international/v1/authentication/checkapikey' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data '{
  "apiKey": "123abc",
}'
```

</RequestExample>

<ResponseExample>

```json Response
{
    "status": 200,
    "data": {
        "version": "PROD1",
        "success": true,
        "message": "Valid API key.",
    }
}
```
</ResponseExample>
